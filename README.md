## **Resumen versión 1**

La primera versión del proyecto reúne los códigos iniciales que hemos utilizado para entender cómo funciona todo.  
La introducción se divide en tres partes:  
- **Clases:** Amelia Pop  
- **Ficheros:** Xian Suárez Duro  
- **Funciones:** Álvaro Fernández Camarzana  

El archivo **airport.py** es la base del proyecto en esta versión, mientras que **interface.py** contiene la primera estructura de la interfaz gráfica.  
El documento **ProyectoAeropuerto.pdf** explica el contexto general del proyecto y las cuatro versiones que lo componen.  
Los archivos **.txt** incluyen los datos de aeropuertos que se cargan al ejecutar el programa.  
Por último, **README.md** es este propio archivo donde se explica todo.

## **Resumen versión 2**

En la segunda versión hemos ampliado el proyecto añadiendo la gestión de vuelos que llegan al aeropuerto de Barcelona (LEBL).  
Ahora no solo trabajamos con aeropuertos, sino también con información de vuelos como su identificador, la compañía aérea, el aeropuerto de origen y la hora de llegada.  

Para ello hemos creado el archivo **aircraft.py**, donde definimos la clase *Aircraft* y todas las funciones necesarias para cargar, guardar y manejar estos datos.  

Además, hemos añadido nuevas funcionalidades para analizar la información de los vuelos, como por ejemplo:
- Ver cuántos vuelos llegan en cada hora del día  
- Mostrar cuántos vuelos hay por aerolínea  
- Comparar vuelos Schengen y no Schengen con gráficos  

También hemos implementado la visualización de los vuelos en Google Earth, mostrando las trayectorias desde el aeropuerto de origen hasta Barcelona.  

Por último, añadimos una función para detectar vuelos de larga distancia (más de 2000 km), lo que permite hacer un análisis más completo.  

La interfaz gráfica se ha ampliado para incluir todas estas nuevas opciones y poder usarlas de forma más cómoda.
